A different mental model

Same skills, a different mental model

PostgreSQL (previous workshop)
  • Rows in fixed-column tables
  • Schema changes need a migration
  • Relationships resolved with JOINs
  • One shape per table, always
MongoDB (this workshop)
  • Documents in flexible collections
  • New shapes need zero migration
  • Relationships: embed or reference
  • Different shapes, same collection
The real problem · Forbes, 2011

Why a magazine rewrote its entire CMS

  • Forbes needed to publish articles, videos, slideshows, and sponsored posts — each shaped completely differently
  • A relational schema meant either one giant table full of NULLs, or a new migration for every new content type
  • They rebuilt the CMS on MongoDB and shipped it in 2 months
“MongoDB’s document model meant developers could build new features quickly, easily incorporate changes, and better handle a growing diversity of data types.”
— MongoDB customer case study: mongodb.com/solutions/customer-case-studies/forbes

File 1 of 4 · 01_crud.js

CRUD basics

  • Read: find(), findOne(), filters, sort, limit, count
  • Write: insertOne / insertMany, then updateOne with $set, $push, $inc
  • No JOIN: resolve author_id yourself, query inside an embedded array
  • Goal isn't the syntax — it's getting comfortable before Exercise 2's real payoff
File 2 of 4 · 02_schema_design.js

Schema design: embed vs. reference

  • Same collection, four shapes — article, video, slideshow, sponsored — zero migrations
  • Embed comments (bounded, always read with the article) vs. reference authors (one author, thousands of articles)
  • slug is the one field every shape keeps fixed — Forbes' own "stable API on fluid data" idea, in miniature
  • $jsonSchema validation: MongoDB is schema-flexible, not schema-less
File 3 of 4 · 03_aggregation.js

Aggregation pipelines

  • find() can't compute "average shares per tag" — that needs $group
  • Top articles by shares, engagement by content type, author stats via $lookup
  • Comment-level analysis with $unwind and $size
  • This is Forbes' own words: "real-time insight into social sharing"
File 4 of 4 · 04_indexes.js

Indexes

  • explain() first — see COLLSCAN become IXSCAN
  • Unique index on slug — two articles can never collide on a URL
  • Compound index (type, published_at) — matches the real "latest by type" query
  • Text index for on-site search, with a relevance score
