From Stage 1 to Stage 2

Same skills, a different mental model

Stage 1 · PostgreSQL
  • Rows in fixed-column tables
  • Schema changes need a migration
  • Relationships resolved with JOINs
  • One shape per table, always
Stage 2 · MongoDB
  • Documents in flexible collections
  • New shapes need zero migration
  • Relationships: embed or reference
  • Different shapes, same collection
The real problem · Forbes, 2011

Why a magazine rewrote its entire CMS

  • Forbes needed to publish articles, videos, slideshows, and sponsored posts — each shaped completely differently
  • A relational schema meant either one giant table full of NULLs, or a new migration for every new content type
  • They rebuilt the CMS on MongoDB and shipped it in 2 months
“MongoDB’s document model meant developers could build new features quickly, easily incorporate changes, and better handle a growing diversity of data types.”
— MongoDB customer case study: mongodb.com/solutions/customer-case-studies/forbes
